# 中国高速路线规划 Skill

智能规划驾车路线，充分利用各地绕城高速免费政策为车主节省过路费。

## 依赖 Skill

**amap-lbs-skill**（高德地图服务）
- amap-lbs-skill 路径：`/opt/data/skills/amap-lbs-skill/`
- API Key 配置在 `/opt/data/skills/amap-lbs-skill/config.json`

## 核心原理：绕城免费 + "同站下再上"

**问题**：高德 API 返回的"最短路线"直接计算全程高速费，不会自动利用绕城免费政策。

**解决思路**：对于有绕城免费政策的城市（如郑州绕城对豫A/豫V免费），在行程中**主动下高速再从同一收费站上**，可以"重置"计费，从而把绕城免费段接入路线，只需付必要的过路费。

**举例**：洛阳→开封（豫V）
- 直线路线：197.8km，**80元**（全程连霍高速）
- 优化方案：洛阳→沟赵站（下）→同站上→开封西（下）→开封万岁山，**64元**（省16元，绕路5.6km）

## 数据文件

```
/opt/data/hermes/china-highway-route/
├── policy-cache.json   # 绕城高速免费政策缓存
└── route-planner.js    # 路线优化规划脚本
```

## 功能特性

- 🎯 **自动绕城优化**：分析直线路线，找出可利用的免费绕城段
- 💰 **费用对比**：直路 vs 优化方案，明确省多少钱、绕多少路
- 📏 **阈值判断**：绕路太多则放弃优化，保留直达路线
- 🔍 **政策查询**：利用 `policy-cache.json` 判断车牌适用的免费城市
- 📱 生成高德地图导航链接

## 触发条件

用户表达以下意图时：
- 询问从 A 地到 B 地的驾车路线
- 询问高速费用、过路费
- 包含"路线"、"怎么走"、"多少高速费"等关键词

## 使用流程

### 第一步：确认车牌信息

首次使用必须询问用户车牌。
已知用户默认车牌：豫V（郑州），郑州绕城高速免费。

### 第二步：获取起终点坐标

```python
import urllib.request, urllib.parse, json

key = "你的AMAP_KEY"
def geocode(address):
    url = f"https://restapi.amap.com/v3/geocode/geo?address={urllib.parse.quote(address)}&output=JSON&key={key}"
    with urllib.request.urlopen(url) as resp:
        data = json.loads(resp.read().decode())
    if data["status"] == "1" and data["geocodes"]:
        return data["geocodes"][0]["location"]
    return None
```

### 第三步：运行路线优化脚本

```bash
cd /opt/data/skills/amap-lbs-skill && node scripts/route-planner.js {起点坐标} {终点坐标} {车牌}
```

**示例**：
```bash
cd /opt/data/skills/amap-lbs-skill && node scripts/route-planner.js 112.405227,34.668000 114.328979,34.786026 豫V
```

脚本会自动：
1. 查询直线路线和费用
2. 分析路线经过的绕城高速段
3. 查询 `policy-cache.json`，判断车牌适用的免费城市
4. 计算"同站下再上"分段路线的费用
5. 对比直路 vs 优化方案，输出结果

**脚本输出格式**：
- `optimized: true` + 分段详情 → 使用优化方案
- `optimized: false` → 直达路线最优，无需优化

### 第四步：判断是否采用优化方案

脚本会明确输出判断结果：
- **省费 > 0 且绕路 < 20km** → 采用优化方案，告知用户分段路线
- **否则** → 直达路线最优，直接返回直路结果

### 第五步：返回结果给用户

给用户清晰的结果，包含：
- 直达路线信息（距离、时间、费用）
- 优化方案（如采用）：分段详情、省多少钱、绕多少路
- 高德导航链接

## ⚠️ 高德 API 的根本局限

**高德 `/v3/direction/driving` 接口不支持车牌参数**，返回的高速费是"标准定价 × 距离"，完全不知道任何车牌专属免费政策（如郑州绕城对豫A/豫V免费）。

**后果**：API 对少林→新密段返回 25 元，实际豫V应付 0 元；对其他涉及绕城免费的路线也会严重高估费用。

**解决方案**：在脚本层面做"后处理"——先让 API 返回结果，再对已知的免费段用知识库覆盖费用。不能依赖 API 的 `tolls` 字段。

### 免费段判断逻辑（route-planner.js 核心）

```javascript
// 已知精确费率：伊阙入口→少林出口 = 21元 / 46.4km
const KNOWN_TOLL_RATE_PRE_SHAOLIN = 21 / 46.4;

// 判断路线 step 是否在少林站之后（进入绕城免费段）
function isAfterShaolin(road, instruction) {
  return road.includes('G1516') || road.includes('S85') ||
         instruction.includes('G1516') || instruction.includes('盐洛高速') ||
         instruction.includes('郑少高速');
}
```

**判断策略**：
- step 道路/指示含 G1516/S85/郑少高速 → 少林→新密区间，免费
- G36/G1516 混行段（如 step12）→ instruction 里提到 G1516 → 按比例拆分（少林前正常收费，之后免费）

## 免费段知识库维护规范

每次发现新的精确免费段，按以下格式记录到 `policy-cache.json`：

```json
{
  "免费段详情": {
    "起止站点": {
      "涉及高速": ["高速名称列表"],
      "站点": ["收费站名"],
      "适用车牌": ["豫A", "豫V"],
      "备注": "关键判断逻辑说明",
      "验证数据": {
        "查询段": "结果",
        "站点坐标": "..."
      }
    }
  }
}
```

**原则**：
1. 记录"起止站点"而非"经过哪些高速"——按站点判断更稳定
2. 记录"验证数据"——实际查过的段，下次直接用，不用重复算
3. 发现免费段跨多种高速时，把每种高速单独列出来

## 注意事项

- `route-planner.js` 依赖 `amap-lbs-skill` 的 `node_modules`（axios）
- 脚本放在 `/opt/data/skills/amap-lbs-skill/scripts/` 下运行（才能找到 `node_modules`）
- 绕城接入点（如下高速的收费站）必须和再上的收费站是**同一站**，否则无法重置计费
- 如果优化方案的绕路距离 > 20km，建议用户直接走直路
- 政策可能调整，以 `policy-cache.json` 中 `searchDate` 为准
