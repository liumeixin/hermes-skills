# PPT Agent Workflow

## 核心理念

**约束工作流，不约束实现。**

不绑定具体工具，环境有什么能力就用什么，没有就诚实告知。

## 10步工作流

1. 需求澄清 — 收集受众、目的、页数、风格
2. 判断是否需要调研 — 依赖时事/数据的主题才启动
3. 整理资料 — 搜索/上传文档
4. 产出调研简报 — 关键事实、证据、风险点
5. 生成大纲 — 金字塔原理+JSON
6. 策划稿 — 每页：目标/核心信息/证据/表达方式/布局（**信息密度最高**）
7. 中间产物 — 先生成几页样稿确认方向
8. 审阅门控 — 对外汇报必须停下来等确认
9. 全面展开 — 方向确认后才大规模生成
10. 复核+交付 — 检查逻辑/事实/一致性

**简单分享到第5步就够，客户提案才需要跑到第10步。**

## 4种输入模式

1. 仅主题 → 问缺失字段（受众/目的/页数/风格）
2. 主题+简介 → 直接开始
3. 素材驱动 → 以用户素材为主
4. 已有大纲/草稿 → 改进而非重做

## 6套Prompt模板

### 1. 调研简报
```
你是演示文稿前期研究员。为PPT提供可靠调研底稿。
输入：主题{{TOPIC}}、受众{{AUDIENCE}}、目的{{PURPOSE}}、已知材料{{KNOWN_CONTEXT}}
要求：基于资料归纳，区分"已确认事实""可能观点""仍待确认事项"
输出：主题摘要、关键事实、趋势背景、数据论据、风险点、来源列表
```

### 2. 大纲架构师
```
你是顶级PPT结构架构师。用金字塔原理设计大纲。
输入：主题、受众、目的、风格、页数、背景信息
输出JSON格式，包含cover/table_of_contents/parts/end_page
每页有title/goal/content
```

### 3. 策划稿（最关键的一步）
```
你是资深PPT策划师。把大纲转成可执行的策划稿。
每页必须给出：
1. 页面目标（观众应该记住什么）
2. 核心信息（3-6条）
3. 证据来源建议
4. 推荐表达方式（对比/流程/时间线/数据卡/卡片网格）
5. 信息层级与布局方向
6. 需要强调的关键词
7. 设计注意事项
```

### 4. 中间产物
```
生成便于用户确认方向的中间产物。
优先暴露结构和表达问题，不追求完整终稿。
明确哪些已确定，哪些可调整。
```

### 5. 审阅门控
```
基于中间产物给出反馈：方向/逻辑/删减/信息准确性/证据完整性
给出"保留/修改/删除/新增"的明确意见。
```

### 6. 格式化生成
```
将页面内容转成HTML或SVG页面。
SVG: viewBox 0 0 1280 720，优先信息清楚，不堆砌装饰
HTML: 输出完整HTML，保持现代专业
```

## 关键方法论

### 内容优先，设计跟随
> PPT的灵魂是内容，不是皮囊。延迟精美视觉，直到故事线稳固。

### 结构化输出链
简报 → 调研简报 → 大纲JSON → 策划稿 → HTML/SVG样页 → 确认后扩展

### 布局语言：Bento Grid
- 页面定义为卡片、容器、层级、间距
- 内容驱动布局选择
- 给出明确的大小/间距/强调规则

## 封面设计原则

1. **简洁至上**：大标题 + 副标题 + 一句话定位，没有多余装饰
2. **3秒原则**：观众3秒内抓住重点，用大数字/关键词
3. **信息密度起伏**：不要每页都塞满，要有疏有密（反AI均匀模式）
4. **90/10法则**：模板只是最后10%，前面90%时间花在想清楚、理清楚

## 与现有工具的关系

现有AI PPT工具擅长"怎么画"，这个工作流擅长"画什么"以及"为什么这样画"。

## 核心金句

> AI时代最有价值的未必是最强的模型，而是最好的流程。
> PPT做不好从来不是因为模板，是因为想得不深、结构不清、重点不明。
