# 设计虱写作系统 Skill

**适用场景**：管理设计虱自媒体文章的完整创作流程（从灵感到发表）
**作者**：醉雪梅花（设计虱）
**工作目录**：`/opt/data/workspace/设计虱-写作系统/`

---

## 核心理念

**类比知识库**：知识库沉淀"过去经验"，写作系统沉淀"未来内容"。两者并行、互相引用、不合并。

**和知识库的关系**：
- 写作时从知识库拉背景知识
- 好的写作洞察可以沉淀到知识库
- 入口分开，避免混乱

## 目录结构

```
设计虱-写作系统/
├── 灵感库/
│   ├── 收集箱/           # 原始素材（人管，Agent只读）
│   │   └── YYYY-MM-DD-HH-MM-简述.md
│   └── 灵感卡片/         # 结构化素材（Agent管）
│       ├── YYYY-MM-DD-卡片标题.md
│       └── README.md     # 卡片索引
│
├── 选题库/
│   ├── pending/          # 待写选题（Agent管）
│   ├── writing/          # 正在写（Agent管）
│   ├── completed/        # 已完成选题（Agent管）
│   └── index.md          # 选题总览
│
├── 稿件库/               # 写作中的草稿（Agent管）
│   └── YYYY-MM-DD-文章标题.md
│
└── 已发表/               # 最终产出（人管）
    └── YYYY-MM-DD-文章标题.md
```

## 人机边界（重要！）

| 目录 | 谁管理 | Agent 行为 |
|------|--------|------------|
| 灵感库/收集箱 | **人** | 只读取，不修改 |
| 灵感库/灵感卡片 | **Agent** | 创建、更新、整理 |
| 选题库 | **Agent** | 创建、更新、移动状态 |
| 稿件库 | **Agent** | 创建、更新 |
| 已发表 | **人** | 人手动放入 |

**规则**：
- 收集箱是素材入口，素材第一时间丢这里
- Agent 只读收集箱，定期整理为灵感卡片
- 已发表由人手动管理，Agent 不碰

## 工作流程

### 1. 素材入口（人）
看到好文章片段、零星灵感、有启发的想法 → 丢进 `灵感库/收集箱/`
- 文件名：`YYYY-MM-DD-HH-MM-简述.md`
- 内容：直接粘贴原文或写下想法，不用整理

### 2. 素材整理（Agent）
定期（每周或有新素材时）整理收集箱：
- 读取收集箱的原始素材
- 创建/更新 `灵感库/灵感卡片/` 下的结构化卡片
- 提炼：核心洞察、可延伸方向、关联性
- 打标签（tags）
- 整理完后可清空或标记已处理的收集箱文件

### 2.5 风格提取（写之前）

**触发条件**：首次写作，或用户说"分析一下我的风格"。

**操作步骤**：
1. 读取 `已发表/` 目录下 3-5 篇最新文章
2. 分析并提炼风格特征，输出 **账号风格档案**
3. 存入 `写作系统/风格档案.md`，后续写作自动引用

**分析维度：**

| 维度 | 提取内容 |
|------|----------|
| 人称视角 | 用"我"还是"我们"？是否直接对读者说话？ |
| 段落节奏 | 平均段落长度？长短句比例？一句话成段的频率？ |
| 句式特征 | 常用转场词？口语化程度？是否有标志性口头禅？ |
| emoji使用 | 频率？位置（段首/段尾/不用）？类型偏好？ |
| 论证方式 | 以个人体验为主？数据为主？类比为主？ |
| 结尾风格 | 收尾方式（行动呼吁/金句/开放式/干脆利落）？ |
| 禁区偏好 | 哪些写法是绝对不碰的？ |

**风格档案格式**（存入 `写作系统/风格档案.md`）：
```markdown
---
type: style_profile
updated: YYYY-MM-DD
source_articles: [文章1, 文章2, 文章3]
---

# 设计虱账号风格档案

## 人称与视角
（分析结果）

## 节奏与句式
（分析结果）

## 语言特征
（口头禅、转场词、情绪表达方式等）

## emoji 使用
（频率、位置、类型）

## 结尾模式
（常用收尾方式）

## 绝对禁区
（从已发表文章反推的禁区）
```

**注意**：
- 风格档案不是死规则，是参考基线
- 写作时如果和"最优写法"冲突，优先保风格一致性
- 每发表 5-10 篇后更新一次风格档案

### 3. 选题提炼（Agent）
从灵感卡片组合中提炼选题：
- 选择角度互补或有关联的卡片
- 创建 `选题库/pending/` 下的选题文件
- 关联来源卡片（inspiration_sources）
- 规划写作大纲

### 4. 写作（Agent）
接到写作任务时：
- 确认选题状态改为 writing
- 从选题关联的灵感卡片中提取素材
- 必要时检索知识库补充背景
- 按 design-shi-article skill 的规范写作
- 输出到 `稿件库/`

### 5. 用户修改（人）
Agent 写完初稿后：
- 人手动修改、调整、配图
- 人决定是否发表

### 6. 发表归档（人）
用户手动：
- 将终稿放入 `已发表/`
- 更新选题状态为 completed
- 填写产出文章链接

### 6.5 发表后处理（Agent）
**触发条件**：用户告知某篇文章已发表

**操作步骤**：
1. 在 `已发表/` 目录找到对应文章（通过标题/日期模糊匹配）
2. 在 `稿件库/` 找到对应的草稿文件（通过标题关键词匹配）
3. 对比两份文件的差异，学习用户的修改逻辑（如：删了哪些内容、加了什么角度、标题怎么改的）
4. 将改动要点简述保存到记忆（方便后续写作时借鉴）
5. 删除稿件库中的草稿文件

**注意**：
- 删除前必须确认两边是同一篇文章（标题/日期高度吻合）
- 如果找不到对应草稿，跳过删除，在回复中说明
- 小红书精简版等备用稿，由用户决定是否丢弃，Agent 不主动保留

**示例**：
```
用户说："XX文章已发表"
→ Agent 在已发表/找到 XX.md，在稿件库找到 XX-草稿.md
→ 对比差异，发现用户删了代码块、加了3张截图、改了结尾
→ 学习并简述改动要点
→ 删除稿件库/XX-草稿.md
```

## 从知识库获取素材

定期从知识库筛选适合转化为写作素材的内容：
- 标记为 `memory_layer: semantic` 且 `confidence` 高的概念
- 来源页面中有独特观点或方法的
- 可以直接在 `收集箱` 中引用或提炼

## 触发词

- "有新素材" / "整理素材" → 整理收集箱
- "有个想法" / "我想写" → 创建灵感卡片或选题
- "开始写" / "写文章" → 进入写作流程
- "定期整理" → 从知识库和收集箱批量整理
