# 设计虱写作系统 Skill

**适用场景**：管理设计虱自媒体文章的完整创作流程（从灵感到发表）
**作者**：醉雪梅花（设计虱）
**工作目录**：`/opt/data/workspace/设计虱-写作系统/`

---

## 核心理念

**类比知识库**：知识库沉淀"过去经验"，写作系统沉淀"未来内容"。两者并行、互相引用、不合并。

**和知识库的关系**：
- 写作时从知识库拉背景知识
- 好的写作洞察可以沉淀到知识库
- 入口分开，避免混乱

---

## 目录结构

```
设计虱-写作系统/
├── 灵感库/
│   ├── 收集箱/           # 原始素材（人管，Agent只读）
│   │   └── YYYY-MM-DD-HH-MM-简述.md
│   └── 灵感卡片/         # 结构化素材（Agent管）
│       ├── YYYY-MM-DD-卡片标题.md
│       └── README.md     # 卡片索引
│
├── 选题库/
│   ├── pending/          # 待写选题（Agent管）
│   ├── writing/          # 正在写（Agent管）
│   ├── completed/        # 已完成选题（Agent管）
│   └── index.md          # 选题总览
│
├── 稿件库/               # 写作中的草稿（Agent管）
│   └── YYYY-MM-DD-文章标题.md
│
└── 已发表/               # 最终产出（人管）
    └── YYYY-MM-DD-文章标题.md
```

---

## 人机边界（重要！）

| 目录 | 谁管理 | Agent 行为 |
|------|--------|------------|
| 灵感库/收集箱 | **人** | 只读取，不修改 |
| 灵感库/灵感卡片 | **Agent** | 创建、更新、整理 |
| 选题库 | **Agent** | 创建、更新、移动状态 |
| 稿件库 | **Agent** | 创建、更新 |
| 已发表 | **人** | 人手动放入 |

**规则**：
- 收集箱是素材入口，素材第一时间丢这里
- Agent 只读收集箱，定期整理为灵感卡片
- 已发表由人手动管理，Agent 不碰

## 工作流程

### 1. 素材入口（人）
看到好文章片段、零星灵感、有启发的想法 → 丢进 `灵感库/收集箱/`
- 文件名：`YYYY-MM-DD-HH-MM-简述.md`
- 内容：直接粘贴原文或写下想法，不用整理

### 2. 素材整理（Agent）
定期（每周或有新素材时）整理收集箱：
- 读取收集箱的原始素材
- 创建/更新 `灵感库/灵感卡片/` 下的结构化卡片
- 提炼：核心洞察、可延伸方向、关联性
- 打标签（tags）
- 整理完后可清空或标记已处理的收集箱文件

### 3. 选题提炼（Agent）
从灵感卡片组合中提炼选题：
- 选择角度互补或有关联的卡片
- 创建 `选题库/pending/` 下的选题文件
- 关联来源卡片（inspiration_sources）
- 规划写作大纲

### 4. 写作（Agent）
接到写作任务时：
- 确认选题状态改为 writing
- 从选题关联的灵感卡片中提取素材
- 必要时检索知识库补充背景
- 按 design-shi-article skill 的规范写作
- 输出到 `稿件库/`

### 5. 用户修改（人）
Agent 写完初稿后：
- 人手动修改、调整、配图
- 人决定是否发表

### 6. 发表归档（人）
用户手动：
- 将终稿放入 `已发表/`
- 更新选题状态为 completed
- 填写产出文章链接

## 索引维护

Agent 负责维护以下索引（自动同步）：

1. `灵感库/灵感卡片/README.md` - 卡片索引（按状态/标签）
2. `选题库/index.md` - 选题总览

---

## 从知识库获取素材

定期从知识库筛选适合转化为写作素材的内容：
- 标记为 `memory_layer: semantic` 且 `confidence` 高的概念
- 来源页面中有独特观点或方法的
- 可以直接在 `收集箱` 中引用或提炼

---

## 触发词

- "有新素材" / "整理素材" → 整理收集箱
- "有个想法" / "我想写" → 创建灵感卡片或选题
- "开始写" / "写文章" → 进入写作流程
- "定期整理" → 从知识库和收集箱批量整理
