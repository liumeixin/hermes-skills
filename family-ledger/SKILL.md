# Family Ledger - 家庭账本

记账技能，全家人可以通过不同渠道使用。

---

## 用户识别

### 用户映射
- 映射文件：`/opt/data/hermes/family-ledger/user-mapping.json`
- 格式：`{"ou_xxx": "显示名", ...}`
- 从当前会话的 openid 查找对应显示名，用于 `recorded_by` 字段

### 未知 openid 处理流程
当 openid 不在 user-mapping.json 中时：
1. **先确认是否 openid 变化**：询问"你之前用过这个账号吗？还是第一次使用？"
2. **排除旧用户换号的情况**：如果用户说"我是 XXX"，检查 user-mapping.json 中是否有同名用户，如有则说明可能是 openid 变了，直接更新映射
3. **确认是新用户后**：询问"你希望怎么称呼？"，得到名字后经用户同意写入 user-mapping.json

### 记账人规则
- **禁止硬编码 payer/recorded_by 名字**：必须从 user-mapping.json 查当前会话 openid，匹配到谁就是谁
- 新建记录时自动填入 `recorded_by`（显示名）

## 账本文件位置

```
/opt/data/hermes/family-ledger/
├── daily.json    # 日常账本
├── social.json   # 人情账本
├── borrow.json   # 借款报销账本
└── user-mapping.json  # 用户映射
```

## 日常账本 (daily.json)

### 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| date | string | 日期，格式 YYYY-MM-DD |
| type | string | 类型：收入 / 支出 |
| amount | number | 金额，单位 元 |
| category | string | 类别 |
| house | string | 房屋（可选） |
| child | string | 孩子（教育类别可选） |
| note | string | 备注 |
| recorded_by | string | 记账人（必填，自动从消息发送者判断） |

### 支出类别

餐饮、交通、购物、工资、房租、水电、通讯、医疗、教育、娱乐、房屋维护、其他

### 收入类别

工资、奖金、投资、兼职、礼金、其他

### 房屋名称

军龙苑、保利大都会、东方文创园、洛铜嘉苑、龙锦嘉园

> 当备注中出现上述房屋名称时，自动归类为"房屋维护"，并在 `house` 字段记录。

## 指令格式

### 记一笔账

**自动判断账本类型**：
- 日常类关键词（餐饮/交通/购物/工资/房租/水电/通讯/医疗/教育/娱乐/房屋维护）→ 日常账
- 人情类关键词（压岁钱/随礼/人情）→ 人情账
- 借款类关键词（借/还/报销）→ 借款账

**时间处理**：
- 用户没说时间 → 默认当天
- 用户说了时间 → 按用户说的

**示例**：

```
记一笔账：50元 餐饮 午餐（自动使用当天日期）
记一笔账：2026-03-20 500 压岁钱 给儿子 张三 张小明
记一笔账：借出 李四 1000 借钱看病
```

**房屋名称确认**：当备注中出现房屋名称时，先确认再记录。确认后可询问："还有其他房产需要录入吗？"

**教育类别孩子确认**：
- 首次记录教育费用时，询问家里孩子数量和昵称
- 已有孩子信息时，直接归类到对应孩子名下

### 查询记录

```
查日常账 [时间]
例如：查日常账 2026年3月
```

```
查人情账 [时间]
例如：查人情账 今年
```

```
查借款账 [状态]
例如：查借款账 未还
```

### 统计汇总

```
统计日常账 [时间] [类别]
例如：统计日常账 2026年3月 餐饮
```

```
统计人情账 [时间]
例如：统计人情账 今年
```
## 注意事项

1. 日期格式统一为 YYYY-MM-DD
2. 金额为数字，单位 元
3. 状态更新需要明确说明
4. 查询时支持模糊匹配（如"彤彤家"会匹配包含"彤彤"的记录）
5. **时间默认值**：用户未指定日期时，自动使用**当前日期**（Asia/Shanghai 时区）
6. **账本类型自动判断**：根据关键词自动路由到对应账本
7. **房屋名称确认**：首次提到房屋名称时需确认，后续自动识别
8. **孩子信息**：教育类费用需要关联到具体孩子
9. **时区**：所有日期计算统一使用 Asia/Shanghai（UTC+8）
